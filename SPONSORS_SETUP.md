# Sponsors Setup

Goal: make it easy for people who value Nkama Fact Benchmark to support ongoing
development.

## What Codex Can Do

- Keep `.github/FUNDING.yml` ready.
- Draft sponsor tier copy.
- Add README sponsor wording after the funding URL exists.
- Verify that the sponsor button appears after GitHub setup is complete.

## What Only KK Can Do

GitHub Sponsors is account-owned. The project owner must:

1. Join GitHub Sponsors.
2. Complete the sponsored developer profile.
3. Create sponsorship tiers.
4. Submit bank or fiscal-host information.
5. Submit tax information.
6. Enable GitHub 2FA.
7. Submit the profile for GitHub approval.

Official setup guide:
`https://docs.github.com/en/sponsors/receiving-sponsorships-through-github-sponsors/setting-up-github-sponsors-for-your-personal-account`

## Suggested Tiers

These are draft labels. Keep the promise modest until the project has users.

### $5/month - Supporter

For people who want evidence-gated AI workflows to stay open source.

Reward:

- Name listed in sponsor thanks, if they want.

### $25/month - Builder

For developers, teachers, researchers, and agent users trying Nkama in real work.

Reward:

- Name or project listed in sponsor thanks, if they want.
- Priority consideration for feature ideas.

### $100/month - Team Sponsor

For teams using Nkama-style evidence manifests in agent workflows.

Reward:

- Logo/name listed in README sponsor section, if they want.
- Monthly short update about roadmap and reliability work.

## FUNDING.yml After Approval

After the GitHub Sponsors profile is approved, use:

```yaml
github: [donkk11]
```

If you use another funding platform first, add the official URL as a custom
funding link:

```yaml
custom: ["https://example.com/your-funding-page"]
```

Do not add fake or inactive funding links. Funding links should point to real
support pages only.
